Learning Outcomes

1. Transfer to the Cloud

The student will get practical experience in how to set up a multi-machine environment that is able to seamlessly run both locally and on a cloud provider.

Referencing modules: Enter the Cloud

Assessed by: Assessment for Entering the Cloud

2. Testing a Cloud Application

The student will gain practical experience of testing a cloud application.

Referencing modules: Implement Cloud Application

Assessed by: Assessment for Project Test Plan

3. Install and Test VirtualBox

The student will be able to create a new virtual machine running Ubuntu.

Referencing modules: Getting Started With Virtualisation

Assessed by: Assessment for Getting Started with Virtualisation

4. Deploy a Cloud Application

The student will gain practical experience of deploying a cloud application.

Referencing modules: Deploy your Cloud Application

Assessed by: Assessment for Project Deployment, Checkpoint: Deployment

5. Setting up a Multi-Machine environment with Vagrant

The student will get practical experience in using Vagrant to set up an environment consisting of several heterogeneous machines.

Referencing modules: Multiple Machines

Assessed by: Assessment for Multiple Machines

6. Understand what provisioning is, and the need for it

The student will gain a basic understanding of what provisioning is, why it is needed, and some of the common tools available to set up your virtual machines.

Referencing modules: Provisioning

Assessed by: Assessment for Provisioning

7. Reasons for using Cloud Computing

The student will gain an understanding of different reasons for using cloud computing. The student will motivate the reasons for using a cloud solution for their specific project.

Referencing modules: The Cloud Business Case

Assessed by: Assessment for Selecting a Project

8. Develop a cloud application architecture

The student will identify architecture relevant decisions and develop an architecture for their cloud application project.

Referencing modules: Cloud Application Architecture

Assessed by: Assessment for Project Architecture, Checkpoint: Architecture Presentation

9. Develop a deployment plan

The student will create a deployment plan that enables local development and testing as well as seamless cloud deployment.

Referencing modules: Cloud Application Architecture

Assessed by: Checkpoint: Architecture Presentation, Assessment for Project Deployment Plan

10. Provisioning with Vagrant

The student will get practical experience in using the built-in capabilities of Vagrant to set up a virtual machine.

Referencing modules: Provisioning

Assessed by: Assessment for Provisioning

11. Select Project

The student will select a project to implementing a cloud solution for. The student will create an initial business case for this project.

Referencing modules: The Cloud Business Case

Assessed by: Assessment for Selecting a Project

12. Lookup Service

The student will get practical experience in how to set up a multi-machine environment that is aware of how to reach the other machines in the configuration.

Referencing modules: Multiple Machines

Assessed by: Assessment for Multiple Machines

13. Reflections on Deploying a Cloud Application

The student will reflect upon the tools used and the challenges when developing, testing, and deploying a cloud application. The student will reflect upon how to monitor the state of the running cloud application, along with how to ensure other quality attributes such as reliability, scalability, security, privacy, and efficiency.

Referencing modules: Deploy your Cloud Application

Assessed by: Checkpoint: Deployment, Deployment Report

14. Implement a Cloud Application

The student will gain practical experience of implementing a cloud application.

Referencing modules: Implement Cloud Application

Assessed by: Assessment for Project Implementation

15. Install and Test Vagrant

The student will be able to set up a basic ubuntu virtual machine with vagrant and ssh into it.

Referencing modules: Getting Started With Virtualisation

Assessed by: Assessment for Getting Started with Virtualisation
